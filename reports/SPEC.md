# GOPOS 餐飲集團雲端報表系統 - 設計規格

## 1. 系統概述

**目標：** 為餐飲集團提供即時、多店鋪的雲端報表系統

**核心功能：**
- 📱 手機優先設計，隨時隨地查看
- 🏪 多店鋪即時營業數據
- 📅 日期範圍靈活選擇
- 📈 圖表走勢分析
- 🔄 即時數據同步

## 4. 功能設計

### 4.1 首頁儀表板
- 總營業額（本週/本月）
- 總訂單數
- 平均客單價
- 同比上期變化百分比

### 4.2 店鋪選擇器
- 下拉式店鋪選擇
- 全選/取消全選
- 最近查看的店鋪
- 收藏常用店鋪

### 4.3 日期範圍選擇
- 今日
- 昨日
- 本週
- 本月
- 自定日期範圍
- 快速日期選擇器

### 4.4 單日營業概覽
- 店鋪名稱
- 今日營業額
- 今日訂單數
- 平均客單價
- 到訪人數

### 4.5 多日趨勢圖
- 折線圖/柱狀圖
- 每日營業額走勢
- 每日訂單數走勢
- 移動平均線
- 最高/最低標記

### 4.6 數據對比
- 與上期比較
- 同期增長率
- 顏色標識（綠色上漲/紅色下跌）

## 6. 技術實現

### 6.1 前端
- React Native 或 Flutter（手機App）
- 或者 PWA 響應式網頁
- Chart.js 或 ApexCharts 圖表庫

### 6.2 後端
- Python Django REST API
- 或者 Node.js Express
- 實時數據 WebSocket（可選）

### 6.3 數據庫
- PostgreSQL（雲端）
- 每日自動同步 GOPOS SQL Server

### 6.4 部署
- Zeabur 雲端服務器
- CDN 加速

## 7. 用戶流程

```
1. 用戶打開報表系統
   ↓
2. 自動登入（記住登入狀態）
   ↓
3. 看到首頁儀表板（總營業額、本週概覽）
   ↓
4. 點擊店鋪選擇器 → 選擇想查看的店鋪
   ↓
5. 選擇日期範圍（今日/本週/自訂）
   ↓
6. 查看營業額、走勢圖、訂單數
   ↓
7. 可以點擊任一天查看當日詳細
   ↓
8. 匯出Excel/PDF（可選）
```

## 8. 報表類型

| 報表名稱 | 內容 | 適用場景 |
|---------|------|---------|
| 日營業報表 | 每日營業額、訂單數、客單價 | 日常管理 |
| 週報表 | 本週趨勢、環比變化 | 週會檢討 |
| 月報表 | 月度總結、同比分析 | 月度報告 |
| 店鋪排名 | 按營業額排序各店表現 | 競賽激勵 |
| 時段分析 | 各時段營業額分佈 | 人力安排 |

## 9. 權限設計
